Что такое вайбкодинг?

Вайбкодинг — это подход к разработке, где ты описываешь что хочешь получить на обычном языке, а ИИ генерирует код. Твоя роль — архитектор идей.

🧠

Ты — идейный вдохновитель

Ты описываешь задачу, формулируешь требования и направляешь ИИ. Понимание концепций важнее синтаксиса.

🤖

ИИ — твой напарник-разработчик

ChatGPT, Claude, Gemini пишут код, исправляют ошибки, объясняют что происходит и предлагают улучшения.

🔁

Итерация — главный навык

Не нужно писать идеально с первого раза. Запустил → посмотрел → уточнил запрос → улучшил. Это и есть процесс.

⚡

7 модулей к твоему первому приложению

Каждый модуль — это практика. Никаких скучных лекций без результата.

01
Среда и первый запуск
Настройка инструментов · 2–3 дня
Старт
▾
  • Установи Android Studio — скачай с developer.android.com. При установке выбери все компоненты по умолчанию. Настрой AVD (эмулятор) — виртуальный телефон на твоём компьютере.
  • Создай первый проект через File → New Project → Empty Activity. Язык: Kotlin. Минимальный SDK: API 26 (Android 8+). Название: «MyFirstApp».
  • Запусти на эмуляторе — нажми зелёную кнопку ▶. Увидишь «Hello World!» на экране телефона. Поздравляю — это уже работающее Android-приложение!
  • Спроси ИИ объяснить структуру проекта
    💬 Промпт для ChatGPT / Claude
    «Я только что создал первый Android-проект на Kotlin. Объясни структуру папок простыми словами: что делает каждый файл, на что обращать внимание новичку? Отвечай кратко, без лишней теории.»
  • Измени текст «Hello World» на своё имя, используя ИИ для помощи.
    💡 Не бойся ломать код. Это самый быстрый способ учиться. ИИ всегда поможет починить.
02
Jetpack Compose — UI без боли
Современный способ делать интерфейсы · 4–5 дней
Основы
▾
  • Что такое Compose: декларативный UI — ты описываешь КАК выглядит экран, а не КАК его нарисовать.
  • Попроси ИИ создать простой экран
    💬 Промпт
    «Создай экран профиля на Jetpack Compose. Нужно: круглый аватар (placeholder), имя пользователя, краткая bio, кнопка "Подписаться" в фиолетовом цвете. Используй Material3. Покажи весь код composable-функции.»
  • Изучи ключевые composables: Text, Button, Image, Column, Row, Box, Modifier. Для каждого попроси ИИ дать пример с объяснением.
  • Практика: список из 5 карточек фильмов.
    @Composable fun MovieCard(title: String, rating: Float) { Card(modifier = Modifier.fillMaxWidth()) { // Попроси ИИ заполнить содержимое! } }
  • State: создай счётчик лайков.
    💬 Промпт
    «Напиши composable-функцию: кнопка с иконкой сердца и счётчик. При каждом нажатии счётчик увеличивается. Объясни зачем нужен remember и mutableStateOf.»
03
Навигация и несколько экранов
Как пользователь перемещается по приложению · 3–4 дня
Навигация
▾
  • Добавь Navigation Compose в build.gradle. Попроси ИИ дать актуальную строку зависимости.
  • Создай 3 экрана: Главный, Детали, Профиль.
    💬 Промпт
    «Напиши навигацию для Android Compose приложения: 3 экрана — Home, Details (принимает itemId: Int), Profile. Нижняя навигационная панель с иконками. Material3. Полный рабочий код.»
  • Передача данных между экранами:
    NavHost(navController, startDestination = "home") { composable("home") { HomeScreen(navController) } composable("details/{id}") { back -> val id = back.arguments?.getString("id") DetailsScreen(id) } }
  • Мини-проект: Каталог книг с детальным экраном.
    💡 Данные о книгах пока можно хранить прямо в коде как список data class объектов.
04
Работа с данными и API
Загрузка реальных данных из интернета · 5–6 дней
Практика
▾
  • Retrofit — библиотека для работы с API. Попроси ИИ объяснить REST API простыми словами.
  • Подключись к бесплатному API: jsonplaceholder.typicode.com
    💬 Промпт
    «Напиши Android Compose приложение: загружает список постов с jsonplaceholder.typicode.com/posts, отображает в LazyColumn. Используй Retrofit + корутины + ViewModel. Покажи все файлы.»
  • Загрузка изображений — библиотека Coil:
    AsyncImage( model = "https://picsum.photos/200", contentDescription = "Photo", modifier = Modifier.size(100.dp).clip(CircleShape) )
  • Проект: Погодное приложение. API: OpenWeatherMap.
    💬 Промпт
    «Создай погодное приложение на Android Compose. Поле ввода города, кнопка поиска. Загружает данные с OpenWeatherMap API. Отображает: температура, описание, иконка, влажность. Красивый UI в тёмных тонах.»
05
Локальная база данных — Room
Сохранение данных на устройстве · 4–5 дней
Данные
▾
  • Room Database — три компонента: Entity (таблица), DAO (запросы), Database.
    💬 Промпт
    «Объясни Room Database для новичка. Пример: Entity для заметки (id, title, text, timestamp), DAO с insert/delete/getAll, Database-класс. Как использовать в ViewModel с Flow?»
  • Проект: Приложение «Заметки».
    @Entity data class Note( @PrimaryKey(autoGenerate = true) val id: Int = 0, val title: String, val content: String, val timestamp: Long = System.currentTimeMillis() )
  • Swipe to Delete: удаление заметки свайпом влево.
    💡 Описывай ИИ не как это реализовать, а что должен чувствовать пользователь: «При свайпе влево карточка краснеет и исчезает».
06
Firebase — бэкенд без кодинга
Авторизация · База данных в облаке · 5–6 дней
Облако
▾
  • Firebase Console: создай проект, скачай google-services.json в папку app/.
    💬 Промпт
    «Покажи пошагово как подключить Firebase к Android-проекту на Kotlin. Какие зависимости добавить в build.gradle?»
  • Firebase Auth: регистрация и вход через email/пароль.
  • Firestore: облачная база данных в реальном времени.
    db.collection("notes") .add(note) .addOnSuccessListener { /* успех */ } .addOnFailureListener { e -> /* ошибка */ }
  • Проект: Социальные заметки — данные синхронизируются между устройствами.
    💡 Google Sign-In добавляется в 10 строк кода. Попроси ИИ — и у тебя есть полноценный вход через Google.
07
Финальный проект и публикация
Сборка APK · Play Console · Портфолио · 5–7 дней
Финал 🚀
▾
  • Выбор идеи для финального приложения.
    💬 Промпт
    «Мне нравится [тема]. Помоги описать MVP Android-приложения. Только самый необходимый функционал за 2 недели. Список экранов и ключевые фичи.»
  • Работа с ИИ над полным проектом. Описывай каждый экран по отдельности. Запускай, тестируй, итерируй.
  • Полировка UI.
    💬 Промпт
    «Вот мой экран [код]. Сделай его визуально привлекательным: улучши типографику, добавь анимации, проверь отступы. Material Design 3.»
  • Сборка APK/AAB: Build → Generate Signed Bundle → создай keystore → подпиши.
  • Google Play Console: $25 единоразово, листинг, скриншоты, загрузка AAB.
    💡 Начни с внутреннего тестирования — бесплатно, без ревью, поделись с друзьями сразу.
  • Портфолио на GitHub с красивым README и гифками. Это твоё доказательство навыков.

4–6 недель до готового приложения

Примерный темп при занятиях 1–2 часа в день.

Неделя 1
Установка и первый экран
Android Studio работает, понимаешь структуру проекта, создаёшь простые UI-компоненты в Compose.
Неделя 2
Многоэкранное приложение
Навигация, списки, карточки, передача данных. Каталог книг или фильмов готов.
Неделя 3
Реальные данные из API
Retrofit настроен, загружаешь данные из интернета, отображаешь картинки. Погодное приложение работает.
Неделя 4
База данных и Firebase
Room сохраняет данные локально, Firebase обеспечивает авторизацию и облачное хранение.
Недели 5–6
Финальный проект → Play Market
Своя идея реализована, приложение опубликовано, портфолио на GitHub, ты — создатель приложений.
